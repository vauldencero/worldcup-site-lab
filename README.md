# worldcup-site-lab

本仓库用于归档和发布多个独立的 HTML 页面，内容涵盖与世界杯相关的信息展示、交互原型、静态演示等。  
该仓库不针对任何特定域名或具体网站，仅作为 HTML 页面资源的集中存放与版本管理用途。

## 项目简介

`worldcup-site-lab` 是一个实验性质的静态页面仓库，主要用来保存、整理和共享与世界杯主题相关的独立 HTML 文件。  
这些页面可以是信息页、模拟页、可视化演示或其他前端原型，彼此之间没有强依赖关系，每个页面均可独立访问。

## 目录说明

```
worldcup-site-lab/
├── pages/           # 存放所有独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/          # 公共资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── docs/            # 文档或说明文件（可选）
└── README.md        # 本文件
```

- `pages/`：每个 HTML 文件代表一个独立的页面，文件名可反映其内容或用途。
- `assets/`：存放页面可能共享的静态资源，便于统一维护。
- `docs/`：用于存放额外的说明、设计稿或引用资料（可选）。

## 页面归档说明

- 所有页面均为纯前端实现，无需后端支持。
- 页面之间互不依赖，可直接在浏览器中打开运行。
- 部分页面可能使用外部资源（如字体、图标库），请确保网络连接正常。
- 归档的页面可能包含实验性或阶段性内容，不保证长期稳定更新。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来改进页面内容、修复错误或添加新页面。
- 新增页面请放入 `pages/` 目录，并推荐在文件名中体现页面主题。
- 公共资源请优先放入 `assets/` 对应子目录，避免重复引入。
- 本仓库不追踪具体的域名或网站，仅关注页面本身的质量与可访问性。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/worldcup-site-lab.git
   ```
2. 直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件。
3. 也可通过 GitHub Pages 或其他静态托管服务发布整个仓库，实现在线访问。
